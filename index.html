Pseudo Elements:

A CSS pseudo-element is a keyword added to a selector that lets you style a specific part of the selected element(s).
For example, it can be used to:

  • Style the first letter, or line, of an element
  • Insert content before, or after, the content of an element

Syntax

                        
                            selector::pseudo-element {
                                property: value;
                            }

Examples

  • first-line: Selects the first line.
  • first-letter: Selects the first letter.
  • before: Insert something before the content.
  • after: Insert something after the content.
  • marker: Selects the markers of list items.
  • selection: Selects the portion of an element that is selected by a user.

Pseudo Classes:

A pseudo-class is used to define a special state of an element.
For example, it can be used to:

  • Style an element when a user mouses over it
  • Style visited and unvisited links differently
  • Style an element when it gets focus

Syntax:

                        
                            selector:pseudo-class {
                                property: value;
                            }

Examples

  • Anchor Pseudo-classes: :link, :visited, :hover, :active
  • :first-child: pseudo-class matches a specified element that is the first child of another element.
  • :last-child: Selects element that is the last child of its parent.
  • nth-child(n): Selects the nth element of its parent.

:is Pesudo Class

                        
                            div:is(a, p, .card, #id) {
                                ...
                            }
    
                            :is(header, .card) p:hover {
                                ...
                            }

Flexbox

A CSS3 layout mode that provides an easy and clean way to arrange items within a container.

  • No floats!
  • Responsive and mobile friendly
  • Positioning child elements is much easier
  • Flex container's margins do not collapse with the margins of its contents
  • Order of elements can be easily changed without editing the source HTML

Flexbox Model Concenpt

  • The ability to alter item width and height to best fit in its containers available free space
  • Flexbox is direction-agnostic, means it works for both horizetal axis and vertical axis
  • Built for small scale layouts while the upcoming "Grid" specification is for more large scale

Grid

Grid allows us to create two dimentional layouts on a webpage, and align items in
both columns and rows.

  • display: gird - Initiates the grid property
  • grid-template-columns: Parent property, give widths to each child to see the effect
  • grid-column-gap (now column-gap): defines the gape between columns. values: any css unit like 1em
  • grid-row-gap: defines the gape between rows. values: any css unit like 1em
  • grid-gap: shorthand for grid-column-gap and grid-row-gap
  • grid-auto-rows: defines the fixed height of the elements in the rows. For flexible height
    use grid-auto-rows: minmax(100px, auto); - By default the height will be 100px but if the content
    goes beyond that height, it will stretch out to fill the content.
  • jsutify-items: Align items horizantally. start | center | end | stretch (default)
  • align-items: Align items vertically. start | center | end | stretch (default)
  • align-self and justify-self: Child property which aligns the child elements both horizantally and vertically.
  • grid-column: Stretches the child element horizantally. e.g: grid-column: 1/3; stretches the element from one to three horizantally.
  • grid-row: Stretches the child element vertically. e.g: grid-row: 1/3; stretches the element from one to three vertically.

In "grid-template-columns" use "fractions" instead of other CSS units as other units you gonna run into margins issues.
1fr 2fr means the second column will be double of the first column.
Examples:
- 1fr 2fr 1fr - First and third column will be same width and the second will be the double
- 1fr 1fr 1fr = repeat(3, 1fr) - Repeat three times 1fr.
- 1fr 2fr 1fr 2fr = repeat(4, 1fr 2fr)

Media Queries

- Allows us to create CSS rules based on various parameters (viewport size, device type etc).
- We can arrange our layout to suite a wide variety of different / screen sizes.

Syntax:

@media <media-type> and (expression) { CSS code }
- media-types: scree | print | TV | speech or all

Example:

- @media TV and (min-width: 1200px) {}
- @media screen and (max-width: 800px) {} - apply the CSS when the screen is 800px and less, if the screen is bigger than 800px then do not apply the rules.
- @media (orientation: landscape/portrait) and (max-width: 700px) {}
- @media (orientation: portrait) {}

Max-Width

Defines the max-width of the media, when we set the media width say "max-width: 900px"
and now if we increase the media size more than 900px, the defined size will remain 900px.

Min-Width

Defines the min-width of the media, when we set the media width say "min-width: 378px"
and now if we decrease the media size less than 378px, the defined size will remain 378px.

No matther if we increase or decrease the size of the media whose maxium width will be 900px
and minimum width will remain 378px.

CSS Units

There are many different units that you can use to define sizes in CSS.

Absolute Units

Absolute units are those that are always the same in any context. px is an absolute unit because the size of a pixel doesnâ€™t change relative to anything else on the page.

Relative Units

Relative units are units that can change based on their context. There are several of them that you want to use them.

Percentage Unit (%):
A percentage that actually based of its parent size. It can be changed according its parent element.
rem and em
em and rem both refer to a font size, though they are often used to define other sizes in CSS. 1em is the font-size of an element (or the element's parent if you're using it to set font-size). So, for example, if an element's font-size is 16px, then setting its width to 4em would make its width 64px (16 * 4 == 64). 1rem is the font-size of the root element (either :root or html). The math works the same with rem as it did with em, but without the added complexity of keeping track of the parent's font size. Relying on em could mean that a particular size could change if the context changes, which is very likely not the behavior you want. rem short for 'root em'. rem : a CSS unit which is relative to the font size of the html element. em : a CSS unit which is relative to the font size of the parent element.
Viewport Units:
The units vh and vw relate to the size of the viewport. Specifically, 1vh is equal to 1% of the viewport height and 1vw is equal to 1% of the viewport width. Thus 100vw will take of the entire width and 50vw obviously it would take of halt the width. These can be useful any time you want something to be sized relative to the viewport, examples including full-height heroes, full-screen app-like interfaces. But the important thing about vw, vh and percentages is that viewport units are based on the entire screen size while percentages are relative to their parent

Transations

Transion enables us to define the transition between two states of an element.
To create a transition effect, you must specify two things:

  • the CSS property you want to add effect to
  • the duration of the effect

Example:

The following example shows a 100px * 100px red <div> element. The <div> element has also specified a transition effect for the width property, with a duration of 2 seconds:

                        
                            div {
                                width: 100px;
                                height: 100px;
                                background: red;
                                transition: width 2s;
                              }

The transition effect will start when the specified CSS property (width) changes value.
Now, let us specify a new value for the width property when a user mouses over the <div> element:

                        
                            div:hover {
                                width: 300px;
                              }

Change Several Property Values

Can be achieved by separting with comma.

                        
                            div {
                                transition: width 2s, height 4s;
                              }

transition-delay

The transition-delay property specifies a delay (in seconds) for the transition effect.
The following example has a 1 second delay before starting:

                        
                            div {
                                transition-delay: 1s;
                              }

transition-duration

Specifies how many seconds or milliseconds a transition effect takes to complete.

transition-property

Specifies the name of the CSS property the transition effect is for.

transition-timing-function

The transition-timing-function property specifies the speed curve of the transition effect.

  • ease - specifies a transition effect with a slow start, then fast, then end slowly (this is default)
  • linear - specifies a transition effect with the same speed from start to end
  • ease-in - specifies a transition effect with a slow start
  • ease-out - specifies a transition effect with a slow end
  • ease-in-out - specifies a transition effect with a slow start and end
  • cubic-bezier(n,n,n,n) - lets you define your own values in a cubic-bezier function

transition

A shorthand property for setting the four transition properties into a single property.

                        
                            div {
                                transition: width 2s linear 1s;
                              }

Animations

An animation lets an element gradually change from one style to another.
You can change as many CSS properties you want, as many times as you want.
To use CSS animation, you must first specify some keyframes for the animation.
Keyframes hold what styles the element will have at certain times.

  • @keyframes Specifies the animation code
  • animation A shorthand property for setting all the animation properties
  • animation-delay Specifies a delay for the start of an animation
  • animation-direction Specifies whether an animation should be played forwards, backwards or in alternate cycles
  • animation-duration Specifies how long time an animation should take to complete one cycle
  • animation-fill-mode Specifies a style for the element when the animation is not playing (before it starts, after it ends, or both)
  • animation-iteration-count Specifies the number of times an animation should be played
  • animation-name Specifies the name of the @keyframes animation
  • animation-play-state Specifies whether the animation is running or paused
  • animation-timing-function Specifies the speed curve of the animation

Example: 

                        
                            /* The animation code */
                            @keyframes example {
                            from {background-color: red;}
                            to {background-color: yellow;}
                            }

                            /* The element to apply the animation to */
                            div {
                            width: 100px;
                            height: 100px;
                            background-color: red;
                            animation-name: example;
                            animation-duration: 4s;
                            }

In the example above we have specified when the style will change by using the keywords "from" and "to" (which represents 0% (start) and 100% (complete)). It is also possible to use percent. By using percent, you can add as many style changes as you like.

Example: 

                        
                            /* The animation code */
                            @keyframes example {
                            0%   {background-color: red;}
                            25%  {background-color: yellow;}
                            50%  {background-color: blue;}
                            100% {background-color: green;}
                            }

                            /* The element to apply the animation to */
                            div {
                            width: 100px;
                            height: 100px;
                            background-color: red;
                            animation-name: example;
                            animation-duration: 4s;
                            }

The animation-direction property specifies whether an animation should be played forwards, backwards or in alternate cycles.The animation-direction property specifies whether an animation should be played forwards, backwards or in alternate cycles.

  • normal - The animation is played as normal (forwards). This is default
  • reverse - The animation is played in reverse direction (backwards)
  • alternate - The animation is played forwards first, then backwards
  • alternate-reverse - The animation is played backwards first, then forwards
